# qrec Demo Video — Storyboard

**Format:** 1280×720 · 30 fps · ~60 seconds · 6 scenes
**Tone:** Clean, developer-native, honest. No fluff. Show the real product.
**Palette:** qrec blue (`#0062a8`) background throughout. White UI panels.

---

## Narrative Arc

> A developer has hundreds of Claude Code sessions — and no way to find anything.
> A tool appears. The chaos organizes itself. Meet qrec.
> One command installs it. The sessions come alive — indexed, enriched, searchable.
> Find anything from the UI. Find anything from the terminal. Nothing is ever lost.
> Come build with us.

---

## Scene 1 — The Problem

The screen is dark blue and empty. Session cards begin tumbling in from every edge — top, sides, corners — like papers dropped on a desk. They land at random angles, overlapping, disordered. A counter climbs: **847 sessions**. There is no structure. No way in.

A magnifying glass — the qrec logo — springs in from the center of the screen. A soft glow rings the lens.

The cards inside the lens straighten up. They snap to a clean vertical list: titles legible, dates aligned, project colors consistent. The cards outside follow suit, collapsing into order. The chaos resolves.

Below the lens, three words fade in:

> **Meet qrec.**

The logo scales up and glides off the top of the screen. From the bottom edge, a terminal window slides up to fill the frame — the first hint of how to get started.

---

## Scene 2 — Installation

The terminal is on screen. A developer types two commands:

```
$ npm install -g @dvquys/qrec
$ qrec serve --daemon
```

The daemon starts. A browser window springs in — the qrec dashboard. A progress bar shows the embedding model downloading. Then indexing begins. The session count climbs from 0 to 50. Enrichment follows — summaries and tags computed on-device. The activity feed scrolls with live updates. By the end, everything is ready.

*One command. It handles the rest — model download, indexing, enrichment, all local.*

---

## Scene 3 — Project Filter

The full dashboard holds. A sessions list slides up. A cursor drifts to the project filter pills and clicks **qrec**. The heatmap and session list both update instantly — now showing only sessions from the `qrec` project.

*Your sessions are organized by project. Filter down to exactly what matters.*

---

## Scene 4 — Enrichment

The cursor moves to a session in the list and clicks. The detail view expands. The title reads as a truncated raw question: *"How can we ensure old sessions stay…"*

An AI-style editing cursor slides in and positions itself at the end of the title. It blinks, thinking. Then it deletes the raw title character by character — a pause — and rewrites it from scratch:

> **Archive JSONL on index for session durability**

Content builds out below, each field typing itself in:
- **Summary** — a concise paragraph of what happened
- **Tags** — `#indexer`, `#durability`, `#archive` — appear as styled badges
- **Learnings** — two durable insights extracted from the conversation
- **Questions answered** — the original question, resolved

*qrec doesn't just store sessions — it reads them. Summaries, tags, and learnings generated by a local model. No API calls. No tokens spent.*

---

## Scene 5 — Search

We open on the enriched session detail. A cursor moves to the NavBar and clicks **Search**. The tab switches. The session detail fades; the search UI appears.

Before the terminal arrives, a small figure — Clawd, the qrec mascot — peeks in from the right edge of the screen. Arms up, excited. It gestures toward the empty right half of the frame, as if to say: *watch this.*

The screen splits. The browser narrows to the left half. The terminal slides in from the right — Clawd fades as the terminal takes its place, as if the mascot *became* the tool.

Both panels come alive simultaneously:
- The browser search bar types: **archive JSONL**
- The terminal types: `$ qrec search "archive JSONL" --k 3`

Results appear progressively in both panels — three sessions, staggered, each pairing a UI card with a terminal result line. Scores, titles, and tags match exactly.

*Search from the UI. Search from the CLI. Same results. Your archive, always queryable.*

---

Blue screen. Clawd springs in center-stage — arms raised, excited. The qrec logo arcs in from the left like a tossed object and lands in Clawd's hand. Clawd bobs gently.

```
$ npm install -g @dvquys/qrec
```

```
github.com/dvquy13/qrec
```
